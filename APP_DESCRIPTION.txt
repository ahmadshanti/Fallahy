────────────────────────────────────────────────────────────────────────────────
0.  WHAT THE APP IS  (one paragraph)
────────────────────────────────────────────────────────────────────────────────

من الأرض is an Arabic-first, right-to-left iOS + Android marketplace that
connects Palestinian families directly to local farmers. A buyer sees fresh
produce + live market vs. farm prices, places an order with a tap, can adopt
a real fruit tree and see AI-generated photos of it in every season, and can
chat with the farmer or an in-app Claude assistant. A farmer dictates new
products by voice (no typing needed), gets an AI revenue forecast and smart
alerts, manages orders, and tracks earnings. Visual identity is olive green
+ cream + a Cairo Arabic display font, with a gold accent for premium /
seasonal touches.

Brand colors
  primary       #2D5016  (deep olive green)
  primaryDark   #1A3A0A
  primaryLight  ~#98C27B
  secondary     mustard/gold
  success       leaf green
  surface       white
  surfaceDim    #F1F5F9
  background    cream-tinted
  AI-accent     #7C3AED  (purple — used on every AI-powered surface)

Typography
  All text uses the Cairo family loaded from Google Fonts:
    Cairo_400Regular    body
    Cairo_600SemiBold   labels, buttons
    Cairo_700Bold       headings, prices
  RTL is forced via I18nManager.forceRTL(true). Flex layouts use
  flexDirection: 'row-reverse' so visual order matches reading order.

Tech stack (so the layout reasoning makes sense)
  Frontend     Expo SDK 54, React Native 0.81, expo-router v6, zustand,
               @tanstack/react-query
  Backend      Supabase (Postgres + auth + storage + RLS + realtime)
  AI service   FastAPI on Render — 9 endpoints (services/ai-service/)
  Image gen    Replicate flux-schnell  (lib/replicate.ts)
  LLM          Anthropic Claude (in the AI service)
  Voice        Azure Speech-to-Text  (in the AI service)
  Maps         Azure Maps with Haversine fallback
  Icons        @expo/vector-icons (Ionicons family throughout)


────────────────────────────────────────────────────────────────────────────────
1.  APP LAUNCH FLOW
────────────────────────────────────────────────────────────────────────────────

Native splash → app/_layout.tsx mounts → app/index.tsx decides where to go:

  • If EXPO_PUBLIC_DEV_MODE=true and not logged in → /(auth)/dev-role
  • Else not logged in                            → /(auth)/onboarding
  • Logged in, role=farmer                        → /(farmer)
  • Logged in, role=buyer                         → /(buyer)

There's also a designed splash screen at /(auth)/splash.tsx (logo + "من الأرض"
fade-in over a 800ms timer on a deep-green background) that's used when the
in-app splash is wanted instead of jumping straight to onboarding.

Nine endpoints, each with a DEMO fallback that returns sensible mock data

  POST  /api/ai/chat                       Claude (buyer/farmer assistant)

  POST  /api/voice/parse-product           Claude → structured Product fields

  GET   /api/analytics/price-comparison    market price vs Fallahy + saving %
  GET   /api/analytics/price-ticker        live day-over-day direction
  GET   /api/analytics/price-trend         time-series for chart screens
  GET   /api/analytics/farmer-dashboard    revenue forecast + smart_alerts

Data files bundled with the service:
  farmers.json   6 real Palestinian/Jordanian farmers with lat/lng
  prices.json    14-day price history for 4 crops

The React Native app talks to this service via lib/aiService.ts. Every UI
that depends on it has a Supabase / mock fallback so the screen never goes
blank when the service is down.

────────────────────────────────────────────────────────────────────────────────
9.  GLOBAL UX NOTES
────────────────────────────────────────────────────────────────────────────────

• Everything is RTL. Buttons read right-to-left, lists flow right-to-left,
  back arrows point right (arrow-forward Ionicon) because tapping it returns
  you "forward in your reading direction".

• Loading uses a centered ActivityIndicator in primary color. Lists never show
  blocking spinners — sections render their shell immediately and the section
  shows its own small loader while data lands.

• Empty states are friendly: muted Ionicon (40-48px) + Arabic message + a
  CTA button when there's a logical next action.

• Errors surface real Supabase/AI error messages instead of generic Arabic
  strings — easier to debug, easier to demo.

• Every screen that writes data has a dev-mode branch that mutates a local
  zustand store instead, so the demo never hits a foreign-key violation from
  the fake dev user UUIDs.

• Native splash is hidden as soon as the JS renders (no font-load wait) so the
  app never appears frozen. The first-render path has a 1.5s safety ceiling
  on the loading spinner.

• Network calls have built-in timeouts: 4s for auth session, 3s for individual
  Supabase queries on the launch path, 5s per section on the buyer home.
  React Query is configured with retry: 1 and 500ms backoff so failures are
  fast.
